<div dir="ltr">We define "empty class" as follows:<div><br></div><div><div>  "A class with no non-static data members other than zero-width bitfields, no virtual functions, no virtual base classes, and no non-empty non-virtual proper base classes."<br></div></div><div><br></div><div>But this doesn't make sense: zero-width bitfields must be unnamed and unnamed bitfields are not members, so there is no such thing as a non-static data member that is a zero-width bitfield. (The same error exists in the definition of "nearly empty class".)</div><div><br></div><div>There is implementation divergence on a case like this:</div><div><br></div><div>  struct A { int : 3; };</div><div>  struct B : A { int n; };</div><div><br></div><div>This type has size 4 in Clang and size 8 in GCC; Clang takes the rule literally, and GCC applies a rule like:</div><div><br></div><div><div>  "A class with no non-static data members, no unnamed bitfields other than zero-width bitfields, no virtual functions, no virtual base classes, and no non-empty non-virtual proper base classes."</div></div><div><br></div><div>As far as I can see, both behaviours are permitted by the C++ standard.</div><div><br></div><div><br></div><div>I'm guessing that GCC's interpretation was the intended one?</div></div>
