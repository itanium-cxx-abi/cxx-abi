<div dir="ltr">Hi,<div><div><br></div><div>The Itanium ABI does not seem to provide a mangling for reference temporaries.<br></div></div><div><br></div><div>Consider the following:<br></div><div><div><div>struct A { const int (&x)[3]; };</div>
<div>struct B { const A (&x)[2]; };</div><div>template <typename T> B &&b = { { { { 1, 2, 3 } }, { { 4, 5, 6 } } } };</div><div>B &temp = b<void>;</div></div><div><br></div><div>The temporaries created by instantiating b<void> must be the same in all translation units.</div>
</div><div><br></div><div>To satisfy this requirement, I propose that we mangle the temporaries in lexical order using a mangling similar to what GCC 4.9 uses and identical to what trunk clang uses.</div><div><br></div><div>
<special-name> ::= GR <object name> [ <nonnegative number> ]<br></div><div><br></div><div>The first special-name would have no trailing number, the second would have '0' and so on allocated in lexical order.</div>
<div><br></div><div>For this example, the following names would be emitted:</div><div>_ZGR1bIvE would be given to the 'B' object that 't' would refer to.<br></div><div>_ZGR1bIvE0 would be given to the array of 'A' object references</div>
_ZGR1bIvE1 would be given to the object containing the first array of ints, {1, 2, 3}<br class="">_ZGR1bIvE2 would be given to the object containing the second array of ints, {4, 5, 6}<div><br></div><div>-- </div><div>David Majnemer</div>
