> For starters, [propagation]p8 does not define how the exception should be<br>
> copied on implementation that do perform this copy.  I presume that you'd<br>
> like it to be copied as if from a const l-value.  However, we do not appear<br>
> to be authorized to introduce a use of such a constructor; [except.throw]p5<br>
> only authorizes the compiler to check access to the constructor that would be<br>
> used to initialize the exception object from the throw operand, and in C++11,<br>
> that will often be a move constructor.<br>
> <br>
> Also, substituting std::bad_exception just because we don’t know how to<br>
> copy the exception, as opposed to because copying the exception actually<br>
> failed, does not actually appear to be authorized.  The standard should define<br>
> the desired behavior when the copy constructor is non-const, inaccessible,<br>
> or defined as deleted.</font></tt><br>
