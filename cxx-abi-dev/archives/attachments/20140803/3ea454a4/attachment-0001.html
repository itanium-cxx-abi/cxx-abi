<div dir="ltr">Alias templates allow the production of new forms of sizeof... for which we have no specified mangling. Example:<div><pre style="color:rgb(0,0,0)">struct A {
  template<typename...T> using N = int[sizeof...(T)];
  template<typename...A, typename B, typename...C>
      void f(N<A..., B, C...> &, B, C...);
};
void g(A a) { int arr[6]; a.f<int, int>(arr, 1, 2, 3, 4); }</pre>No implementation I have access to produces anything reasonable here:</div><div><br></div><div>Clang currently produces a bogus RA3_i mangling for the type of the first parameter.</div>
<div>GCC produces <span style="color:rgb(0,0,0)">RAstDpT__i, that is, sizeof-type applied to pack expansion applied to template-parameter-0, which seems to be exposing a detail of their internal representation, and is in any case wrong since T_ is not enough to specify which packs are relevant.</span></div>
<div><span style="color:rgb(0,0,0)">EDG rejects.</span></div><div><span style="color:rgb(0,0,0)"><br></span></div><div><span style="color:rgb(0,0,0)">Also of note: through alias templates, arbitrary template argument expressions can be incorporated into such a sizeof... expression (as can an implicit test that multiple packs are the same length), so it's not sufficient to merely encode a sequence of packs whose sizes should be added, along with an additional constant for packs of known size.</span></div>
