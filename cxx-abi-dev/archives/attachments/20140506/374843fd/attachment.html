> To implement support for mangling reference temporaries:</font></tt><br>
<tt><font size="2">> <br>
> 1. An additional <special-name> non-terminal production should be added:</font></tt><br>
<tt><font size="2">> <br>
> <special-name> ::= GR <object name> [ <seq-id> ] _   # Reference temporaries</font></tt><br>
<tt><font size="2">> <br>
> The <seq-id> is strictly the lexical order in which the reference <br>
> temporary was written in the source.</font></tt><br>
<tt><font size="2">> <br>
> The following exists as a practical example:</font></tt><br>
<tt><font size="2">> <br>
> _ZGR1bIvE_ would be given to the 'B' object that 't' would refer to.</font></tt><br>
<tt><font size="2">> _ZGR1bIvE0_ would be given to the array of 'A' object references</font></tt><br>
<tt><font size="2">> _ZGR1bIvE1_ would be given to the object containing the first array <br>
> of ints, {1, 2, 3}</font></tt><br>
<tt><font size="2">> _ZGR1bIvE2_ would be given to the object containing the second array<br>
> of ints, {4, 5, 6}</font></tt><br>
<tt><font size="2">> <br>
> 2. The text describing <seq-id> should probably refrain from <br>
> mentioning substitutable entities.</font></tt><br>
